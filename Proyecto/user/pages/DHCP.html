Defensa contra Servidores Falsos (Spoofing)

Bloquea servidores DHCP ilegítimos que intentan entregar configuraciones falsas (como DNS maliciosos) a sus usuarios.

Utilizamos DHCP Snooping, ARP Protect e IP Source Lockdown.

Arquitectura de Alta Disponibilidad (Failover)

Un servidor DHCP caído significa una red caída. NetConfig Pro garantiza la continuidad del servicio mediante la **conmutación por error (Failover)**: configure dos servidores DHCP para que, si uno falla, el otro tome el control al instante.

Ejemplos de Configuración Rápida

Haga clic en cualquier escenario para ver un ejemplo de configuración.

Ejemplo 1: Crear un Ámbito (Scope) Básico en Linux 
# Ejemplo de /etc/dhcp/dhcpd.conf

subnet 192.168.1.0 netmask 255.255.255.0 {
  range 192.168.1.100 192.168.1.200;
  option routers 192.168.1.1;
  option domain-name-servers 8.8.8.8, 1.1.1.1;
  option domain-name "miempresa.local";
  
  default-lease-time 600;
  max-lease-time 7200;
}
Ejemplo 2: Reservar una IP (Asignación Estática por MAC) 
# Añadir esto dentro o fuera de la definición de la subred

host servidor-impresion {
  hardware ethernet 00:1A:2B:3C:4D:5E;
  fixed-address 192.168.1.50;
  option host-name "Impresora-Piso1";
}
Ejemplo 3: Configurar Opciones (DNS/Gateway) en Windows Server 
# En Windows Server, esto se hace a través de la interfaz gráfica:

1. Abra la consola "DHCP".
2. Expanda su servidor > IPv4 > Su Ámbito (Scope).
3. Clic derecho en "Opciones de Ámbito" y "Configurar Opciones...".
4. Marque las siguientes casillas y añada los valores:
   - 003 Router:   Añada la IP de su gateway (ej. 192.168.1.1).
   - 006 Servidores DNS: Añada las IPs DNS (ej. 8.8.8.8).
   - 015 Nombre de Dominio DNS: "miempresa.local".
5. Haga clic en "Aplicar".
Ejemplo 4: Excluir un Rango de IPs (Para Servidores) 
# En Windows Server:

1. Abra la consola "DHCP".
2. Expanda su servidor > IPv4 > Su Ámbito (Scope).
3. Clic derecho en "Grupo de Direcciones" (Address Pool).
4. Seleccione "Nuevo Intervalo de Exclusión...".
5. Introduzca el rango de IPs que desea reservar para asignación manual.
   (Ej. Inicio: 192.168.1.2, Fin: 192.168.1.49).
6. Haga clic en "Añadir".

*Nota: Esto evita que DHCP asigne IPs que usted usa para Servidores.*
